## Logging

All server-side logs are structured JSON via `src/lib/logger.ts`.

**Format:**
```json
{
  "level": "error",
  "message": "Chat stream error",
  "timestamp": "2026-07-14T12:00:00.000Z",
  "env": "production",
  "requestId": "req_m1abc2_def345",
  "userId": "usr_xxx",
  "error": "Connection timeout"
}
```

**Levels:** `debug`, `info`, `warn`, `error`, `audit`

**PII Redaction:** Emails, JWTs, database URLs, and 20+ sensitive keys are automatically redacted.

**Usage:**
```typescript
import { logger } from "@/lib/logger";

logger.info("Operation completed", { userId, duration });
logger.error("Operation failed", { error: err.message });
```

## Request Correlation

Every request gets a unique `x-request-id` header generated by the middleware.

**Propagation:** Header â†’ Next.js middleware â†’ API routes â†’ error responses.

**Response headers:**
- `x-request-id`: Unique request identifier
- `x-response-time`: Server response time in ms

## Analytics (PostHog)

Privacy-friendly analytics via PostHog (self-hostable).

**Environment variables:**
- `NEXT_PUBLIC_POSTHOG_KEY` â€” PostHog project API key
- `NEXT_PUBLIC_POSTHog_HOST` â€” PostHog instance URL (default: `https://us.i.posthog.com`)

**Events tracked:**
- `user_signed_up` (email/github)
- `user_logged_in` (email/github)
- `knowledge_base_created`
- `documents_uploaded`
- `ai_chat_requested`
- `$pageview` (automatic)

**Disable:** Leave `NEXT_PUBLIC_POSTHOG_KEY` unset.

On server start, the application validates:
- Required environment variables
- Database connectivity
- AI provider configuration

Failures are logged at ERROR level; warnings at WARN level.

## Alert Recommendations

| Alert | Threshold | Severity |
|-------|-----------|----------|
| 5xx error rate | > 1% of requests | Critical |
| Health endpoint unhealthy | Any | Critical |
| Database connection failure | Any | Critical |
| Auth failure spike | > 10 in 5 min | High |
| AI provider failures | > 5 in 5 min | High |
| Rate limit abuse | > 100 hits/min | Medium |
| Response time p95 | > 2000ms | Medium |
| Memory usage | > 80% | Medium |
